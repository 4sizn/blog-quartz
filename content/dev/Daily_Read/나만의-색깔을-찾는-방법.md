## 들어가며

우리는 매일 무언가를 하고 있습니다. GitHub에 커밋하고, Notion에 메모하고, 블로그에 글을 쓰고, 새로운 기술을 공부합니다.

하지만 가끔 이런 질문이 떠오릅니다:

> **"내가 지금 하고 있는 활동이 진정 의미있는 것인가?"**

정돈되지 않은 메모들, 완성되지 않은 프로젝트들, 시작만 하고 끝내지 못한 공부들... 우리의 아카이브는 이런 흔적들로 가득합니다.

## 아카이브의 목적

### 왜 우리는 기록하는가?

많은 사람들이 GitHub, Notion, 블로그 등에 자신의 활동을 기록합니다. 그 본래의 목적은 무엇일까요?

**미래의 자신에게 건네주기 위한 아카이브**

우리는 지금 배운 것, 경험한 것, 생각한 것을 미래의 나에게 전달하기 위해 기록합니다. 하지만 많은 경우:

- 정돈되지 않는 메모 정도의 글이 다수
- 체계없이 쌓여가는 컨텐츠
- 방향성 없는 활동의 흔적들

### 진짜 문제

아카이브 그 자체가 문제가 아닙니다. **진짜 문제는 방향성의 부재**입니다.

우리는 종종 묻습니다:
- 내가 지금 하고 있는 활동이 진정 의미있는 것인지
- 이것이 정말 내가 원하는 방향인지
- 나는 어디로 가고 있는 것인지

## 외부적 Motivation의 함정

### 외부적 동기의 예시

- "다른 개발자들도 다 블로그를 하니까 나도 해야지"
- "GitHub 잔디를 심어야 취업에 유리하다더라"
- "요즘 핫한 기술이니까 배워야지"
- "팔로워 수를 늘려야 인정받을 수 있어"

### 왜 문제인가?

외부적 동기로 시작한 활동은:
- **지속하기 어렵습니다**: 동기가 사라지면 활동도 멈춥니다
- **만족감이 없습니다**: 목표를 달성해도 공허함을 느낍니다
- **방향성이 없습니다**: 트렌드에 따라 표류합니다
- **진정성이 없습니다**: 남에게 보여주기 위한 활동이 됩니다

## 나를 위한 마음가짐

### 핵심 질문

**외부적인 Motivation을 배제한 나를 위한 마음가짐이 중요합니다.**

자신에게 물어보세요:

1. **나는 어떤 사람인가?**
   - 내가 진정으로 좋아하는 것은 무엇인가?
   - 내가 자연스럽게 시간을 보내는 활동은?
   - 돈을 받지 않아도 하고 싶은 일은?

2. **나는 어떤 사람이 되고 싶은가?**
   - 5년 후 나는 어떤 모습이길 원하나?
   - 내가 존경하는 사람의 어떤 점을 닮고 싶나?
   - 나만의 독특한 가치는 무엇인가?

### 진정한 동기 찾기

외부가 아닌 내부에서 동기를 찾으세요:

**외부적 동기**:
- GitHub 스타 수
- 블로그 방문자 수
- 취업에 유리한 스펙
- 남들의 인정

**내부적 동기**:
- 문제를 해결하는 즐거움
- 새로운 것을 배우는 기쁨
- 누군가에게 도움이 되는 만족감
- 나만의 것을 만드는 성취감

## 나만의 색깔 찾기

### 1. 자기 관찰

**일주일 동안 자신을 관찰하세요**:
- 어떤 활동을 할 때 시간 가는 줄 모르는가?
- 어떤 주제에 대해 자발적으로 더 알고 싶어지는가?
- 어떤 일을 할 때 에너지가 생기는가?
- 어떤 순간에 가장 나다운 느낌이 드는가?

### 2. 패턴 발견

**과거의 활동을 되돌아보세요**:
- 지금까지 완료한 프로젝트들의 공통점은?
- 오래 지속한 활동들의 특징은?
- 포기한 것들과 계속한 것들의 차이는?

### 3. 가치 정의

**나만의 가치를 정의하세요**:

예시:
- **깊이**: 여러 가지를 얕게 보다는, 하나를 깊게 파고들기
- **실용성**: 이론보다는 실제로 사용 가능한 것 만들기
- **공유**: 배운 것을 다른 사람과 나누기
- **창의성**: 기존에 없던 새로운 것 만들어내기

### 4. 방향성 설정

**구체적인 방향을 정하세요**:

나쁜 목표:
```
"좋은 개발자가 되고 싶다"
"많은 것을 배우고 싶다"
"성공하고 싶다"
```

좋은 방향성:
```
"프론트엔드 성능 최적화 전문가가 되고 싶다"
"개발자들이 실제로 사용할 수 있는 도구를 만들고 싶다"
"복잡한 개념을 쉽게 설명하는 기술 블로거가 되고 싶다"
```

## 실천 방법

### 1. 필터링하기

모든 기회에 YES라고 하지 마세요.

**자신에게 물어보세요**:
- 이것이 나의 방향성과 일치하는가?
- 이것이 나를 원하는 모습으로 가까워지게 하는가?
- 외부의 기대가 아닌 내적 동기로 하는 것인가?

NO라면 과감히 거절하세요.

### 2. 깊이 파고들기

선택과 집중:
- 10개를 시작하고 완료하지 않는 것보다
- 1개를 깊이있게 완성하는 것이 낫습니다

### 3. 일관성 유지

**나만의 색깔은 일관성에서 나옵니다**:
- 같은 스타일의 글쓰기
- 특정 분야에 대한 지속적인 탐구
- 일관된 가치관의 표현

### 4. 정기적인 회고

**분기마다 자신에게 물어보세요**:
- 지난 3개월 동안 내가 한 활동들이 나의 방향성과 일치했는가?
- 외부의 기대에 흔들리지 않았는가?
- 나는 원하는 모습에 가까워지고 있는가?

## 실전 예시

### Case 1: 개발자 A

**방향성 없을 때**:
- React 공부하다가 Vue도 배워야할 것 같아서 시작
- 블로그 글 3개 쓰다가 YouTube도 해야할 것 같아서 시작
- 알고리즘 문제 풀다가 시스템 디자인도 공부 시작
- 모든 것이 중간에 멈춤

**방향성을 찾은 후**:
- "나는 React 생태계의 전문가가 되고 싶다"
- React에 집중, 심화 학습
- React 관련 블로그 글 꾸준히 작성
- React 오픈소스에 기여
- 1년 후 React 분야 전문가로 인정받음

### Case 2: 개발자 B

**방향성 없을 때**:
- 트렌드 따라 이것저것 배움
- GitHub 잔디 심기에만 집중
- 팔로워 수에 신경 씀
- 공허함과 번아웃

**방향성을 찾은 후**:
- "나는 개발자들의 생산성을 높이는 도구를 만들고 싶다"
- CLI 도구 개발에 집중
- 실제로 유용한 패키지 몇 개를 완성
- 적은 사용자지만 진심어린 감사 메시지
- 의미있는 만족감

## 나만의 색깔을 가진다는 것

### 나만의 색깔이란?

- 다른 사람과 구별되는 **독특한 특징**
- 일관되게 유지되는 **정체성**
- 나를 나답게 만드는 **핵심 가치**

### 색깔이 있는 사람의 특징

1. **방향성이 명확합니다**
   - 어디로 가는지 알고 있습니다
   - 선택의 기준이 있습니다

2. **일관성이 있습니다**
   - 하는 일에 연결고리가 있습니다
   - 시간이 지나도 정체성이 유지됩니다

3. **진정성이 느껴집니다**
   - 외부의 기대가 아닌 내적 동기로 움직입니다
   - 자신만의 이야기가 있습니다

4. **깊이가 있습니다**
   - 특정 분야에 대한 전문성
   - 표면적이지 않은 이해

## 결론

### 핵심 메시지

**외부적인 Motivation을 배제한 나를 위한 마음가짐이 중요합니다.**

- **어떤 사람인지**: 자기 이해
- **어떤 사람이 되고 싶은지**: 방향성 설정

### 오늘부터 시작하기

1. **30분 시간 내기**
   - 조용한 곳에서 자신과 대화하세요
   - "나는 누구인가?"를 진지하게 물어보세요

2. **일주일 관찰하기**
   - 어떤 활동에 에너지를 느끼는지 기록하세요
   - 패턴을 발견하세요

3. **방향성 한 문장으로 쓰기**
   - "나는 __________ 한 사람이 되고 싶다"
   - 구체적으로 작성하세요

4. **필터링 시작하기**
   - 다음 기회가 왔을 때, 방향성과 일치하는지 확인하세요
   - 일치하지 않으면 거절하세요

### 마지막 말

아카이브가 가득 차는 것보다 중요한 것은 **방향성있게 쌓여가는 것**입니다.

남들이 인정하는 것보다 중요한 것은 **내가 의미있다고 느끼는 것**입니다.

많은 것을 하는 것보다 중요한 것은 **나다운 것을 하는 것**입니다.

**나만의 색깔을 찾아가세요. 그 여정 자체가 성장입니다.**
